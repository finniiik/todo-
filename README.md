# Todoist Clone

Учебный проект клона Todoist, выполненный в рамках практики по веб-разработке. Приложение помогает планировать дела: создавать задачи и проекты, отмечать выполненные пункты, фильтровать список, искать по тексту и работать со сроками. Данные синхронизируются с Firebase Firestore в реальном времени, а если конфигурация Firebase не задана, приложение продолжает работать на локальном хранилище браузера.

## Стек

Frontend: React 18, Vite 5, Context API, SCSS, react-icons, date-fns.
Backend / БД: Firebase Firestore (без собственного backend-сервера).


## Локальный запуск

```
npm install
npm run dev
```

Сборка production-версии:

```
npm run build
npm run preview
```

Приложение по умолчанию открывается на http://localhost:5173.

## Переменные окружения

Ключи Firebase вынесены в переменные окружения и не попадают в репозиторий. Создайте файл `.env` в корне проекта со следующими полями (значения берутся в Firebase Console, раздел Project settings → General → Your apps → SDK setup):

```
VITE_FIREBASE_API_KEY=
VITE_FIREBASE_AUTH_DOMAIN=
VITE_FIREBASE_PROJECT_ID=
VITE_FIREBASE_STORAGE_BUCKET=
VITE_FIREBASE_MESSAGING_SENDER_ID=
VITE_FIREBASE_APP_ID=
```

Если файл `.env` отсутствует, приложение автоматически переключается на хранение задач и проектов в `localStorage` браузера.

## Деплой

Проект задеплоен как статическое SPA. Команда сборки `npm run build`, директория сборки `dist`. Перед публикацией в настройках хостинга нужно добавить те же переменные окружения, что и в `.env`.

Ссылка на деплой: https://todo-d-kb.vercel.app/


## Реализованная функциональность

* Создание, редактирование (двойной клик), удаление задач.
* Отметка задачи выполненной с синхронизацией статуса в Firestore.
* Дата выполнения у задачи, метки «Сегодня» и «Завтра», подсветка просроченных.
* Поиск по тексту задачи.
* Фильтры списка: все, активные, выполненные.
* Очистка всех выполненных задач одной кнопкой.
* Статистика: всего, выполнено, осталось.
* Управление проектами в боковой панели: добавление, удаление, сохранение между сессиями.
* Светлая и тёмная темы интерфейса с сохранением выбора в `localStorage`.
* Полностью русифицированный интерфейс.

## Архитектура

Приложение построено по компонентной модели React. Корневой компонент `App` оборачивает дерево в `TasksProvider`, который инкапсулирует всю работу с задачами, проектами и Firestore. Представление разделено на layout-компоненты (`Header`, `Sidebar`, `Content`) и функциональные (`Tasks`, `AddTask`, `Checkbox`, `Projects`, `AddProject`, `IndividualProject`). Работа с датами вынесена в `src/helpers/date.js`, конфигурация Firebase изолирована в `src/firebase.js`.

```
[Browser]
   |
   v
[React App]  <-- Context API (TasksProvider) -->  [Firebase Firestore]
                                                    |
                                                    v
                                              [collection: tasks]
```

## ERD

```
+----------------+        +---------------------+
|   projects     |        |        tasks        |
+----------------+        +---------------------+
| projectId  PK  |        | id          PK      |
| name       str |        | text        str     |
+----------------+        | completed   bool    |
                          | dueDate     string  |
                          | createdAt   number  |
                          +---------------------+
```

Проекты хранятся локально (`localStorage`), задачи — в Firestore (или `localStorage` при отсутствии конфигурации).

## Use Case

1. Пользователь добавляет новую задачу с датой выполнения. Приложение сохраняет её в Firestore и сразу отображает в списке.
2. Пользователь отмечает задачу выполненной кликом по чекбоксу. Статус синхронизируется с Firestore.
3. Пользователь применяет фильтр «Выполненные» и нажимает «Очистить выполненные». Все такие задачи удаляются.
4. Пользователь добавляет новый проект через кнопку Add Project в боковой панели. Проект появляется в списке и сохраняется между сессиями.
