# ERD (модель данных Firestore)

Firestore — документоориентированная БД, но связи между сущностями всё равно есть: задачи ссылаются на проект и владельца. Ниже приведена логическая ER-модель.

```
┌─────────────────────┐         ┌──────────────────────┐
│       User          │         │       Project        │
│─────────────────────│         │──────────────────────│
│ uid (PK)            │ 1     N │ projectId (PK)       │
│ email               │─────────│ name                 │
│ displayName         │         │ userId (FK -> User)  │
└─────────────────────┘         │ createdAt            │
          │ 1                   └──────────────────────┘
          │                                │ 1
          │                                │
          │ N                              │ N
          ▼                                ▼
┌──────────────────────────────────────────────────────┐
│                       Task                           │
│──────────────────────────────────────────────────────│
│ id (PK)                                              │
│ task        (string)   — текст задачи                │
│ date        (string)   — дата DD/MM/YYYY             │
│ projectId   (FK -> Project)                          │
│ userId      (FK -> User)                             │
│ archived    (boolean)  — выполнена / в архиве        │
└──────────────────────────────────────────────────────┘
```

## Описание сущностей

- **User** — аккаунт Firebase Auth. Идентификатор `uid` используется как внешний ключ во всех документах.
- **Project** — пользовательский список. Базовые проекты (`INBOX`, `TODAY`, `NEXT_7`) виртуальные и не хранятся в БД. Кастомные проекты лежат в `localStorage`.
- **Task** — основная сущность. Хранится в коллекции `tasks` Firestore. Связь с проектом — по строковому идентификатору `projectId`.

## Коллекции Firestore

| Коллекция | Документ                       | Поля                                              |
|-----------|--------------------------------|---------------------------------------------------|
| `tasks`   | автогенерируемый `id`          | `task`, `date`, `projectId`, `userId`, `archived` |
