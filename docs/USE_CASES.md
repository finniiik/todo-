# Use Case диаграмма и сценарии

```
                ┌──────────────────────────┐
                │     Todoist Clone        │
                │                          │
   ┌──────┐     │  ( Добавить задачу )     │
   │      │─────│  ( Завершить задачу )    │
   │ User │─────│  ( Удалить задачу )      │
   │      │─────│  ( Создать проект )      │
   │      │─────│  ( Фильтровать по дате ) │
   │      │─────│  ( Переключить тему )    │
   └──────┘     │                          │
                └──────────────────────────┘
```

## Сценарий 1. Создание задачи

1. Пользователь выбирает проект в боковой панели.
2. Нажимает «Add Task».
3. Вводит текст и (опционально) дату.
4. Нажимает «Add Task» в форме.
5. Задача сохраняется в Firestore и сразу появляется в списке.

## Сценарий 2. Завершение задачи

1. Пользователь видит список задач выбранного проекта.
2. Кликает по чекбоксу слева от задачи.
3. У задачи проставляется `archived: true`, она исчезает из активного списка.

## Сценарий 3. Создание собственного проекта

1. Пользователь в сайдбаре раскрывает раздел «Projects».
2. Жмёт «Add Project».
3. Вводит название и подтверждает.
4. Проект появляется в списке и доступен для выбора при создании задачи.

## Сценарий 4. Переключение темной темы

1. Пользователь жмёт иконку настроек в шапке.
2. Выбирает пункт «Dark Mode».
3. Всё приложение (шапка, сайдбар, контент, поля ввода) переключается в тёмную палитру; выбор сохраняется в `localStorage`.
